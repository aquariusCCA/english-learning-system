# 使用說明（唯一規則手冊）

> [!IMPORTANT]
> **本檔是全庫規則的唯一真相來源。**
> 一條規則只寫在一個地方：結構與流程在本檔、標籤細節在 [[標籤系統]]、筆記格式在 `範本 Templates/`。其他檔案（CLAUDE.md、各 README）只放連結指向這裡，**不寫摘要、不複寫**——發現重複就刪掉留連結。

目標：**系統化複習＋方便日後檢索**。教材可多套（文法、單字、閱讀、聽力…都能收）。

## 1️⃣ 結構：一套教材＝一資料夾

```
English Learning System/
├── README.md                 ← 總導航（Home / 書櫃），純導航不放規則
├── CLAUDE.md                 ← 給 Claude 的指針（指向本檔）＋目前狀態
├── 00 使用說明與範本/          ← 共用：本檔、標籤系統、範本
├── 01 圖表解構英文文法/         ← 一套教材＝一資料夾（「書」是統稱）
│   ├── README.md             ← 全書目錄＋快速回查
│   ├── 索引/                 ← 「這本書」的索引：主題地圖 MOC、圖表索引
│   ├── 0X 章名/               ← 每章：README＋各主題筆記＋images/
│   ├── Excalidraw/           ← （視需要）手繪大圖原始檔
│   └── 練習/                 ← （視需要）AI 出題的練習卷（規則見 4️⃣）
├── 02 謝孟媛英文文法/           ← 第二套教材（初級／中級分冊＋索引/）
└── 99 索引 Indexes/           ← 「跨書」全域索引：易錯點總表、複習排程
```

- **「書」是統稱**：書本、Notion 課程等，一套教材都算一「書」。
- **「書」「章」由路徑承載**：一篇筆記屬於哪本書、哪一章，看它住的資料夾就知道——frontmatter **不設 `book`／`chapter` 欄位**（避免和路徑存兩份、人工對齊）。只有住在書資料夾**外**的內容（`99` 的跨書索引）才需要標「書」（見下）。
- **新增教材**：開新資料夾 `03 xxx/`，內含章節（或分冊）資料夾＋`索引/`（＋視需要 `Excalidraw/`）。
- **索引擺哪**：
  - 整理「**書**」的索引（主題地圖 MOC、圖表索引）→ 跟著書走，放**該書 `索引/`**。
  - 整理「**你**」的索引（易錯點總表、複習排程）→ 跨書共用，放頂層 **`99 索引 Indexes/`**，並加一欄「**書**」標明每筆來自哪套教材。

## 2️⃣ 寫一篇筆記

> [!NOTE]
> **兩條軸，先分清楚**
> - **內容類型**（文法／單字／閱讀…）決定**用哪份範本**：文法 → [[文法主題範本]]（目前唯一的內容類型範本；日後各類型有自己的範本）。
> - **來源**（書本截圖／既有筆記如 Notion）決定**轉入方式**（2-1／2-2）與忠實度的對象，**不決定用哪份範本**。
>   （例：《圖表解構》〔書本〕與謝孟媛〔Notion〕同屬文法類，都用文法主題範本。）

**流程：**
1. **確認來源**——書本截圖走 2-1、既有筆記走 2-2。**來源不清楚就先問使用者，不要猜。**
2. 從 `範本 Templates/` 複製對應**內容類型**的範本（另有工具型範本：易錯點卡片、章節 README）。
3. 存到該書對應章節資料夾，檔名 `序號 主題.md`（序號照該書／課程順序），例：`01 主詞＋不及物動詞.md`。
4. 填 frontmatter：`page` **或** `source`（依來源擇一，見 3️⃣）、`difficulty`、`status`（見 4️⃣）、`tags`（規則見 [[標籤系統]]）、`related`（可留空）。**不填 `book`／`chapter`**——書與章由路徑承載（見 1️⃣）。
5. 依範本區塊填內容，**全程遵守 3️⃣ 忠實度原則**。
6. **轉入後回源核對**：對照來源逐項檢查——例句完整（無 `…` 殘句）、中譯保留、無整段缺漏；核對通過才算轉入完成。
7. 回該章 `README.md`，把這篇加進「小節導航」表並更新狀態。

> 💡 Obsidian 可把 `範本 Templates` 設為 Templates 外掛的範本資料夾，一鍵套用。

### 2-1 來源是書本（課本截圖）
- 提供該小節的**課本截圖**；照截圖**忠實轉錄**概念／例句／表格／注意事項，書中圖表**重繪**成 Mermaid 或 Excalidraw。
- frontmatter 記 `page`（頁碼，可回書對照）。
- ⚠️ **原始課本截圖不進版控**（版權＋保持 repo 乾淨）：用完刪除，或放 gitignore 的暫存夾。

### 2-2 來源是既有筆記（如 Notion）
- **預設忠實保留你原本的整理、不改寫**——你當初整理的內容就是「來源內容」。
- 使用者**明示啟用**時，單篇（或批次）可改走「**教學型重構**」模式：事實不變、敘事重寫，定義與忠實度界線見 3️⃣ 的 3-1。
- 來源語法**轉成相容寫法**（見 5️⃣）：callout → GitHub alerts、折疊 → `<details>`、結構圖 → Mermaid。
- **圖片**：Notion 等的圖是**會過期的臨時網址** → 下載到該章 `images/` 再嵌入。
- frontmatter 記 `source:`（來源連結）。

## 3️⃣ 忠實度原則（來源內容 vs AI 補充）
- **內容預設忠實轉錄自來源**：概念、例句、表格、注意事項以來源為準——來源可為**書本原文**，或你**既有的整理筆記**（如 Notion）。
- **凡來源沒有、由 AI 補充的**（額外易錯點、自測題、延伸提醒），小節標題一律加「**💬 AI 補充**」，一眼區分來源 vs AI。
- **殘句與句內補字**（各模式通用）：筆記或來源出現殘句（帶 `…`）時，**先回來源查證**，查到完整句直接替換（原句屬來源內容，不加標記）；僅當**來源本身就是殘句**、查無完整句時才可補完，補上的字句一律**用〔〕括起**——〔〕是句內級的 AI 標記，與區塊級的「💬 AI 補充」對應。
- **範本是彈性檢查表，雙向不硬套**：
  - 來源**沒有**的區塊（沒圖、沒易錯…）→ 直接**省略，或填但標 💬**。**絕不為了填滿版面而捏造。**
  - 來源**有**、範本**沒有**對應區塊 → **自行新增區塊保留**（標題自訂，例：`## 📎 例外／補充`）。範本是「最小起點」不是上限或篩子，**內容絕不因範本沒欄位就丟掉**。
- 「一句話核心」是對來源內容的濃縮重述。
- 「🧭 心智圖」區塊同理，屬對本篇內容的**濃縮重述**（整塊不標 💬）：節點與例句**一律取自正文、不得新造**；正文修改後心智圖須同步。（區塊格式與生成規則見 [[文法主題範本]]；語法與工具支援見 5️⃣。）
- **出處記錄**：書本來源填 `page`、既有筆記來源填 `source`（擇一），方便日後查證忠實度。

### 3-1 教學型重構（可選模式，僅限 2-2 既有筆記來源）
- **啟用**：由使用者**明示指定**（單篇或批次）才可用；未指定一律走預設忠實轉錄。
- **忠實度對象從「字句」改為「事實」**：所有文法點、規則、例句仍須**出自來源、不得捏造**，保留 `source` 供逐項回查；但章節結構、講解順序、推導敘事可由 AI 重寫——把來源的結論倒推成「從問題出發、先講為什麼」的講解。
- **標記界線**：串接來源事實的講解敘事（動機、推導、記法）屬重構本身，**不逐句標 💬**；來源查證不到的**新增文法規則、例句、易錯點、自測題**照舊標「💬 AI 補充」。
  - **❌ 教學反例**：由來源例句**變形**而來、用於展示錯誤寫法的 ❌ 句，屬講解敘事，不標 💬；全新自創的**正確**例句才標。
- **聲明**：frontmatter 加 `style: 教學型重構`，表明此篇字句經過重組、忠實度以事實層為準。

## 4️⃣ 狀態與複習

frontmatter `status` 的定義：

| 狀態 | 意義 |
| --- | --- |
| 未讀 | 尚未開始 |
| 學習中 | 已建立、正在整理內容 |
| 已複習 | 完成至少一輪間隔複習 |
| 已精熟 | 自我測驗全對、能主動舉例與辨析 |

**間隔複習**：每讀完一篇，把複習日排進 [[複習排程]]（間隔日期見該檔）；**怎麼複習見 [[複習方法]]**（以主動回想為主）。
- 每讀完一章，去 [[易錯點總表]] 把該章重點易錯點集中檢視一次。

**練習與錯題回寫**：
- 請 AI 出的練習卷放**該書 `練習/`** 資料夾，整份標「💬 AI 補充」，並註明取材自哪幾篇筆記。
- 做完後**答錯的題目必回填**：把**錯誤模式**（❌／✅／為什麼錯，非整道題目）寫回出題來源筆記的「**⚠️ 易錯點分析**」區塊（非來源內容，標 💬）。**章節易錯點求全、總表求精**——一條易錯點**反覆犯**才「升級」收進 [[易錯點總表]]；總表是跨書的**錯題精華**，由練習結果餵養，**不是各章易錯點的完整鏡像**，也不憑印象整理。
- 章節 README「學習驗收」的「錯題已回填」勾選項，指的就是這件事（書中練習與 AI 練習卷皆適用）。

## 5️⃣ 跨工具相容寫法
筆記要在 **GitHub／Obsidian 原生**顯示正常；**VS Code 需另裝擴充**（已列在 `.vscode/extensions.json`，開此資料夾時會提示一鍵安裝）。

| 需求 | 寫法 | 工具支援 |
| --- | --- | --- |
| 提示框 | **GitHub-style alerts**（`> [!NOTE]`／`[!TIP]`／`[!IMPORTANT]`／`[!WARNING]`，大寫；標題放內文粗體）。**不要用 Obsidian 專屬 callout**（`[!abstract]` 等小寫） | GitHub／Obsidian 原生；VS Code 裝 `yahyabatulu.vscode-markdown-alert` |
| 可折疊（如自測解答） | HTML `<details><summary>…</summary>…</details>` | 全部原生 |
| 結構圖 | **Mermaid**（` ```mermaid `） | GitHub／Obsidian 原生；新版 VS Code 內建（舊版裝 `bierner.markdown-mermaid`） |
| 心智圖 | **Markmap**（` ```markmap ` 內寫巢狀清單，格式見 [[文法主題範本]]） | Obsidian 裝社群外掛 **Mindmap NextGen**；VS Code 裝 `phoihos.markdown-markmap`（已列入 `.vscode/extensions.json`）；GitHub 不渲染、退化為程式碼框內的縮排大綱（內容仍可讀——此為刻意接受的例外） |
| 手繪大圖 | **Excalidraw** 匯出 SVG 存 `images/`，`![說明](images/檔名.svg)` 嵌入；原始檔放該書 `Excalidraw/` | 全部原生（SVG） |
| `[[雙向連結]]` | 一般筆記用 wikilink | Obsidian 原生；VS Code 裝 **Foam**（`foam.foam-vscode`） |
| 連**根 README（總導航）** | 一律相對連結 `[總導航](../README.md)`（`../` 依深度）——vault 內多個 README，bare `[[README]]` 會指錯；帶資料夾的 `[[章名/README|…]]` 則明確、可用 | — |
| 自動索引 | **不用 Dataview**（僅 Obsidian 支援）；索引一律手動維護 | — |

> 想要最完整體驗：用 Obsidian「Open folder as vault」開這個資料夾。

## 6️⃣ 檢索（日後怎麼找回來）
- **靠標籤**：`文法/時式`、`#易錯點`…（規則見 [[標籤系統]]）。
- **靠索引**：各書 `索引/` 的主題地圖 MOC、圖表索引（名稱依教材而定）；全域的 [[易錯點總表]]（考前）、[[複習排程]]。
- **靠 README**：該書 README 有「快速回查」；章節 README 有「小節導航」。
- **靠檔名序號**：第一次順讀、複習時跳讀。
