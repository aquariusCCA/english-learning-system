# 使用說明

這份筆記系統的目標是：**系統化複習 + 方便未來檢索**。目前教材有兩套：《LiveABC 主編學堂　圖表解構英文文法》（書本）、謝孟媛英文文法（自學筆記，整理自 Notion）；未來可再擴充其他英文教材（單字、閱讀、聽力…）。

## 🗂️ 資料夾怎麼分
```
English Learning System/
├── README.md                 ← 總導航（Home / 書櫃）
├── 00 使用說明與範本/          ← 共用：規則、標籤、範本
├── 01 圖表解構英文文法/         ← 一套教材＝一資料夾（範例；「書」為統稱）
│   ├── README.md             ← 全書目錄 + 快速回查
│   ├── 索引/                 ← 「這本書」的索引：主題地圖 MOC、圖表索引
│   ├── 0X 章名/               ← 每章：README + 各主題筆記 + images/
│   └── Excalidraw/           ← （視需要）手繪大圖原始檔
├── 02 謝孟媛英文文法/           ← 第二套教材（初級／中級分冊 + 索引/）
└── 99 索引 Indexes/           ← 「跨書」的全域索引：易錯點總表、複習排程
```

> 📌 **索引擺哪的原則**：整理「**書**」的索引（主題地圖、圖表索引）→ 跟著書走、放該書資料夾；整理「**你**」的索引（易錯點總表、複習排程）→ 跨書共用、留在頂層 `99`（因跨書，索引內加一欄「**書**」標明每筆來自哪本教材）。

## ✍️ 記一篇新筆記的流程
1. **先確認來源**：書本截圖 → 見下方「📷 從書本（課本截圖）轉入」；既有筆記／Notion → 見「🔄 從既有筆記（Notion）轉入」。**來源不清楚時先問使用者，不要亂猜。**
2. 從 `範本 Templates/` 選一份**對應內容類型的範本**複製（**內容類型範本目前只有 文法主題範本**；另有 易錯點卡片、章節 README 等工具型範本。不同內容類型用各自的範本，見下方 🧩）。
3. 放進該書對應的章節資料夾，檔名用 `序號 主題`（序號對齊該書／課程的順序），例：`01 主詞＋不及物動詞.md`。
4. 填 frontmatter：`chapter`、`page`／`source`（依來源）、`difficulty`、`status`、`tags`。
5. 依**範本的區塊**填內容（文法主題範本：📊 圖表解構 → 🎯 核心觀念 → 📐 用法規則 → ✏️ 例句 → ⚠️ 易錯點），**全程遵守下方「📖 忠實度原則」**（忠實轉錄來源、AI 補充標 💬、範本雙向彈性；細節見該節，不在此重複）。
6. 回到該章 `README.md`，把該篇加進「小節導航」表並打勾。

> 🧩 **範本依「內容類型」而定，不是依來源**：文法類用 `文法主題範本`；日後單字、閱讀、聽力、寫作等各有自己的範本（不同區塊）。**來源（書本截圖／Notion）是另一條軸**——它決定 workflow（📷／🔄）與範本套用的鬆緊，不決定用哪份範本。（例：圖表解構〔書本〕與謝孟媛〔Notion〕同為文法類，都用 `文法主題範本`。）

> 💡 在 Obsidian 可把 `範本 Templates` 設為 Templates 外掛的範本資料夾，一鍵套用。

## 📖 忠實度原則（來源內容 vs AI 補充）
- **內容預設忠實轉錄自來源**：概念、例句、表格、注意事項以**來源**為準——來源可為**書本原文**，或你**既有的整理筆記**（如 Notion）。
- **凡來源沒有、由 AI 補充的內容**（額外易錯點、自測題、延伸提醒），小節標題一律加 **💬 AI 補充**，讓你一眼區分「來源 vs AI」。
- **範本是彈性檢查表，不硬套**：來源**有**的區塊才填；來源**沒有**的區塊（如某點沒圖、沒易錯）→ 直接**省略，或填但標 💬**。**絕不為了填滿版面而捏造來源沒有的內容。**
- **反過來，來源有、範本沒有對應區塊 → 一律保留、自行新增區塊**：範本是「最小起點」不是「上限」或「篩子」；來源內容**絕不因為範本沒欄位就丟掉**，找不到合適區塊就新增一個（標題自訂，例：`## 📎 例外／補充`）。
- **一句話核心**為濃縮摘要，屬對來源內容的重述。
- **出處記錄**：書本來源用 `page`（可回書對照）；既有筆記／Notion 來源用 `source`（記來源連結）——兩者依來源擇一填，方便查證忠實度。

## 📷 從書本（課本截圖）轉入
當來源是**書本**時（例：《圖表解構英文文法》）：
- 提供該小節的**課本截圖**給我；我**忠實轉錄**概念／例句／表格／注意事項，書中圖表**重繪**成 Mermaid／Excalidraw。
- ⚠️ **原始課本截圖不進版控**（版權＋保持 repo 乾淨）：用完刪除，或放進 gitignore 的暫存夾。
- frontmatter 用 `page` 記頁碼；AI 另補的內容標 **💬 AI 補充**。

## 🔄 從既有筆記（如 Notion）轉入
當來源是你**既有的整理筆記**（例：Notion 的「謝孟媛文法」）而非一本書時：
- **忠實保留你原本的整理、不改寫**；你當初整理的內容就是「來源內容」。
- 來源的語法**轉成相容寫法**（callout → GitHub alerts、折疊 → `<details>`、結構圖 → Mermaid）。
- **圖片**：Notion 等的圖是**會過期的臨時網址** → 下載進該筆記的 `images/` 再嵌入。
- frontmatter 用 `source:` 記來源連結；AI 另補的內容一樣標 **💬 AI 補充**。

## 🔁 複習節奏（間隔複習）
每讀完一篇，把複習日排進 [[複習排程]]，建議間隔：**第 1 天 → 3 天 → 7 天 → 14 天 → 30 天**。
- 複習時：先看 **一句話核心**（`> [!IMPORTANT]` 提示框），再做「🧠 自我測驗」，最後掃「⚠️ 易錯點」。
- 每讀完一章，去 [[易錯點總表]] 把該章重點易錯點集中檢視一次。

## 🏷️ 狀態定義（frontmatter `status`）
| 狀態 | 意義 |
| --- | --- |
| 未讀 | 尚未開始 |
| 學習中 | 已建立、正在整理內容 |

| 已精熟 | 自我測驗全對、能主動舉例與辨析 |

## 🖥️ 用什麼工具看（相容性）
筆記採「跨工具相容寫法」：**GitHub／Obsidian 原生顯示；VS Code 需另裝幾個擴充**（已列在 `.vscode/extensions.json`，開此資料夾時 VS Code 會提示一鍵安裝）：
- **提示框**用 GitHub-style alerts（`> [!NOTE]`／`[!WARNING]`／`[!IMPORTANT]`／`[!TIP]`）；**可折疊**內容（如自測解答）用 HTML `<details>`。⚠️ **VS Code 內建預覽不會渲染 alert**（只顯示成引用區塊）→ 需裝 `yahyabatulu.vscode-markdown-alert`（Markdown Preview for Github Alerts）；GitHub／Obsidian 原生支援。
- **`[[雙向連結]]`**：VS Code 需裝 **Foam** 擴充（`foam.foam-vscode`）才能點擊，並有關係圖與反向連結（近似 Obsidian 體驗）；Obsidian 原生。
- **連根 README（總導航）**：vault 內有多個 `README.md`，bare `[[README]]` 會指錯 → 一律用相對連結 `[總導航](../README.md)`（`../` 依檔案深度）；帶資料夾的 `[[章名/README|…]]` 則明確、正常。
- **圖表**：結構圖用 **Mermaid**（GitHub 原生；**新版 VS Code 已內建**、舊版需裝 `bierner.markdown-mermaid`；Obsidian 原生）；手繪風大圖用 **Excalidraw** 匯出 SVG 嵌入（見 `01 圖表解構英文文法/Excalidraw/`）。
- **Dataview**：本庫索引已改**手動維護、不放 Dataview 區塊**（跨工具都乾淨）；Dataview 只有 Obsidian 支援，你若想自己加自動查詢再用。
- 想要最完整體驗，用 Obsidian「Open folder as vault」開這個資料夾即可。
