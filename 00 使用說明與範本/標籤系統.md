# 標籤系統（Tag 規範）

標籤是最快的橫向檢索方式，服務**整個筆記庫（跨所有書）**。全部使用巢狀標籤（`大類/子類`）。

> [!IMPORTANT]
> **使用原則**
> - **先登記、再使用**：要用新的**核心標籤**（主分類 namespace、功能標籤）時，**先在本檔登記一筆，再到筆記 `tags:` 使用**——避免造出同義詞（如 `文法/時態` vs `文法/時式`）讓檢索破碎。臨時、一次性的標籤可隨手加。
> - **擠了才加深，不預先切細**：某個子分類累積到約 **15 篇以上**，才在它底下開第三層（例：`文法/詞類/介系詞`）——照樣先登記、並回補該桶舊筆記。巢狀標籤的**父層自動涵蓋子層**，加深不會弄壞既有檢索。反之別提前切到一標籤只對應一篇——那是檔名的複述，屬冗餘。

## 1. 主分類：依「內容類型」開 namespace
每篇筆記**必加一個**所屬類型的主分類。目前只有 `文法/*` 使用中；日後新內容類型（單字、閱讀、聽力…）依同樣模式開自己的 namespace（如 `單字/*`），**並在此登記**。

> [!TIP]
> **每個 namespace 的父層就是它的總標籤**
> 巢狀標籤的父層（`#文法`…）自動涵蓋其下所有子標籤。要一次抓「全部文法筆記」用 `#文法` 即可，不需另設英文總標籤。

## 2. 功能標籤（依需要疊加）
| 標籤 | 用途 |
| --- | --- |
| `#易錯點` | 這篇（或這段）是易錯重點，會被 [[易錯點總表]] 收集 |
| `#圖表` | 內含值得回查的重繪圖表，會被 [[圖表索引]] 收集 |
| `#對比辨析` | 兩個以上易混觀念的比較（如：現在完成 vs 過去簡單） |
| `#待補` | 內容還沒整理完 |
| `#句型公式` | 有可背誦的結構公式（文法類專用） |

> 類型專用的功能標籤（如日後單字類的 `#搭配詞`）需要時登記在上表並註明類型。

## 3. 結構標籤（系統／索引檔用，非內容）
| 標籤 | 用途 |
| --- | --- |
| `home` | 總導航首頁（根 README） |
| `moc` | 目錄／內容地圖頁（總導航、主題地圖） |
| `章節索引` | 章節 README |
| `全域` | 跨書的全域檔（本檔、易錯點總表、讀完登記…） |
| `meta` | 系統說明檔 |

## 4. 命名原則
- 一律小寫英文或中文，不加空格；階層用 `/`。
- 主分類**只加一個**（該類型的 `類型/*`）；功能標籤可多個。
- 不確定分類時先加 `類型/待分類` + `#待補`，之後再歸位。

## 範例
```yaml
tags:
  - 文法/時式        # 主分類，必加一個
  - 對比辨析         # 功能標籤，依需要疊加
  - 易錯點
```
