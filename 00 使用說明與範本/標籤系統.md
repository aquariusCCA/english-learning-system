# 標籤系統（Tag 規範）

標籤是最快的橫向檢索方式，服務**整個筆記庫（跨所有書）**。**規則越一致，日後越好找**。全部使用巢狀標籤（`大類/子類`）。

> [!IMPORTANT]
> **使用原則：先登記、再使用**
> 要用新的**核心標籤**（主分類 namespace、功能標籤）時，**先在本檔登記一筆，再到筆記 `tags:` 使用**——避免造出同義詞（如 `文法/時態` vs `文法/時式`）讓檢索破碎。臨時、一次性的標籤可隨手加。

## 1. 主分類：依「內容類型」開 namespace
每篇筆記**必加一個**所屬類型的主分類。不同**內容類型**（文法／單字／閱讀…）各開自己的巢狀 namespace：

| 類型 | 主分類 namespace | 狀態 |
| --- | --- | --- |
| 文法 | `文法/*` | ✅ 使用中 |
| 單字 | `單字/*` | ⬜ 日後有單字書再定義 |
| 閱讀 | `閱讀/*` | ⬜ 日後再定義 |
| 聽力 | `聽力/*` | ⬜ 日後再定義 |

> [!TIP]
> **每個 namespace 的父層就是它的總標籤**
> 因為是**巢狀標籤**，父層（`#文法`、`#單字`…）會自動涵蓋其下所有子標籤。要一次抓「全部文法筆記」用 `#文法` 即可（不需另設 `grammar` 之類的英文總標籤）。

## 2. 功能標籤（依需要疊加）
### 通用（跨所有類型）
| 標籤 | 用途 |
| --- | --- |
| `#易錯點` | 這篇（或這段）是易錯重點，會被 [[易錯點總表]] 收集 |
| `#圖表` | 內含值得回查的重繪圖表，會被 [[圖表索引]] 收集 |
| `#對比辨析` | 兩個以上易混觀念的比較（如：現在完成 vs 過去簡單） |
| `#待補` | 內容還沒整理完 |

### 類型專用（只在該類型出現）
| 類型 | 標籤 | 用途 |
| --- | --- | --- |
| 文法 | `#句型公式` | 有可背誦的結構公式 |
| （單字） | 例：`#搭配詞`、`#字根` | 日後有單字書再定義 |

> 其他類型之後可加自己的專用功能標籤，補在這一節即可。

## 3. 結構標籤（系統／索引檔用，非內容）
| 標籤 | 用途 |
| --- | --- |
| `moc` | 目錄／內容地圖頁（總導航、主題地圖） |
| `章節索引` | 章節 README |
| `全域` | 跨書的全域檔（本檔、易錯點總表、複習排程…） |
| `meta` | 系統說明檔 |

## 4. 命名原則
- 一律小寫英文或中文，不加空格；階層用 `/`。
- 主分類**只加一個**（該類型的 `類型/*`）；功能標籤可多個。
- 不確定分類時先加該類型的 `類型/待分類` + `#待補`，之後再歸位。

## 範例
```yaml
tags:
  - 文法/時式        # 主分類（文法類）；其他類型結構相同，如 單字/多益
  - 對比辨析
  - 易錯點
```
