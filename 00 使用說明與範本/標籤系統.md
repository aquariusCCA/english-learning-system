# 標籤系統（Tag 規範）

標籤是最快的橫向檢索方式。**規則越一致，日後越好找**。全部使用巢狀標籤（`大類/子類`）。

## 1. 主分類（每篇必加一個 `文法/*`）
> [!TIP]
> **`#文法` 就是總標籤**
> 因為是**巢狀標籤**，父層 `#文法` 會自動涵蓋所有 `文法/*` 子標籤。
> 要一次抓「全部文法筆記」時，直接用 `#文法`（Obsidian 搜尋與 Dataview `FROM #文法` 皆適用），**不需要**另設 `grammar` 之類的英文總標籤。

對應本書 18 章，收斂成幾個大主題：

## 2. 功能標籤（依需要疊加）
| 標籤 | 用途 |
| --- | --- |
| `#易錯點` | 這篇（或這段）是易錯重點，會被 [[易錯點總表]] 收集 |
| `#圖表` | 內含值得回查的重繪圖表，會被 [[圖表索引]] 收集 |
| `#句型公式` | 有可背誦的結構公式 |
| `#對比辨析` | 兩個以上易混觀念的比較（如：現在完成 vs 過去簡單） |
| `#待補` | 內容還沒整理完 |

## 3. 命名原則
- 一律小寫英文或中文，不加空格；階層用 `/`。
- 主分類**只加一個**；功能標籤可多個。
- 不確定分類時先加 `文法/待分類` + `#待補`，之後再歸位。

## 範例
```yaml
tags:
  - 文法/時式
  - 對比辨析
  - 易錯點
```
