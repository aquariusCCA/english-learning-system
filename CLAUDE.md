# CLAUDE.md — English Learning System 筆記庫

英文學習筆記庫（可多本書）。目前教材：《LiveABC 主編學堂　圖表解構英文文法》（18 章）。
使用者用**繁體中文**、主要在 **VS Code** 檢視；本地 git 庫（分支 `main`）。

> [!IMPORTANT]
> **動手做／改筆記前，先讀 `00 使用說明與範本/` 這三份，並照它們的規則做：**
> - `使用說明.md`（流程、忠實度原則、相容性、檢索、索引擺放）
> - `標籤系統.md`（標籤規範——**先登記再用**）
> - `範本 Templates/`（選對應**類型**的範本；文法用 `文法主題範本.md`）

## 核心規則（摘要，細節以 `00/` 為準）
- **多書結構**：一本書＝一資料夾（`01 圖表解構英文文法/…`），內含 章節資料夾 + `索引/`（該書 MOC、圖表索引）+ `Excalidraw/`。
- **忠實度**：內容**忠實轉錄書本**（以該書原文為準）；書本沒有、由 AI 補充的（自測題、額外易錯、延伸）→ 小節標題加「**💬 AI 補充**」。**範本區塊是彈性的：書本沒有的區塊就省略或標 💬（不硬填、不捏造）；書本有、範本沒有對應區塊 → 新增區塊保留（不丟書本內容）。範本是最小起點、非上限。**
- **跨工具相容寫法**（VS Code / GitHub / Obsidian 都要能顯示）：
  - 提示框用 **GitHub-style alerts**（`> [!NOTE]`/`[!TIP]`/`[!IMPORTANT]`/`[!WARNING]`，大寫，標題放內文粗體）；可折疊用 HTML `<details>`。**不要用 Obsidian 專屬 callout**（`[!abstract]` 等小寫）。
  - 圖表：結構圖用 **Mermaid**；手繪大圖用 **Excalidraw** 匯出 SVG 嵌入。
  - 連「根 README（總導航）」用**相對連結** `[文字](../README.md)`（bare `[[README]]` 會歧義，有 21 個 README）；帶資料夾的 `[[章名/README|…]]` 才用 wikilink。
- **標籤**：主分類依書的**類型**開 namespace（`文法/*`、日後 `單字/*`…）＋通用功能標籤；**新核心標籤先登記到 `標籤系統.md` 再用**。
- **索引擺放**：整理「**書**」的（主題地圖 MOC、圖表索引）→ 放該書 `索引/`；整理「**你**」的（易錯點總表、複習排程）→ 放頂層 `99 索引 Indexes/`，加「書」欄位。
- **原始課本截圖不進版控**（用完可刪或放 gitignore 的暫存夾）。
- **Dataview 僅 Obsidian 有效**：索引改為手動維護，勿再加 Dataview 區塊。

## 從截圖做一篇筆記的流程
1. 讀截圖 → 複製 `範本 Templates/文法主題範本.md`，放進對應章資料夾，檔名 `序號 主題.md`（序號對齊書中小節）。
2. 填 frontmatter（`chapter`、`page`、`difficulty`、`status`、`tags`）。
3. 依範本區塊填：📊（Mermaid）→ 🎯 → 📐 → ✏️ → ⚠️；**書本沒有的區塊就省略或標 💬，不硬填、不捏造**。
4. 更新該章 `README.md` 小節導航（打勾）；重點易錯補進 `99 索引 Indexes/易錯點總表.md`（標來源書）。
5. `git commit`，訊息清楚（如 `docs: 完成第2章 時式 現在完成式`）。
